    bruno-memory¶

    Overview¶

    bruno-memory is a high-performance, multi-backend memory system providing persistent storage and intelligent retrieval for AI conversations and context management.

    Key Features¶

    • üîå Multiple Backends: SQLite, PostgreSQL, Redis, ChromaDB, Qdrant
    • üöÄ High Performance: Connection pooling, caching, batch operations
    • üîç Semantic Search: Vector embeddings with ChromaDB/Qdrant
    • üíæ Flexible Storage: From lightweight SQLite to enterprise PostgreSQL
    • üîÑ Smart Context: Intelligent conversation context building
    • üìä Analytics: Built-in analytics and performance tracking
    • üõ°Ô∏è Type Safe: Full type hints and Pydantic validation
    • üîß Extensible: Plugin architecture for custom backends

    Quick Example¶

    import asyncio
from bruno_memory import create_backend
from bruno_core.models import Message, MessageRole, MessageType

async def main():
    # Create a SQLite backend
    backend = create_backend("sqlite", database_path="memory.db")
    await backend.connect()

    # Store a message
    message = Message(
        content="Hello, how can I help you today?",
        role=MessageRole.ASSISTANT,
        message_type=MessageType.TEXT
    )
    await backend.store_message(message)

    # Retrieve conversation history
    messages = await backend.retrieve_messages(limit=10)
    for msg in messages:
        print(f"{msg.role}: {msg.content}")

    await backend.disconnect()

asyncio.run(main())

    Why bruno-memory?¶

    üéØ Purpose-Built for AI¶

    Unlike generic databases, bruno-memory is specifically designed for AI conversation management with built-in:

    • Conversation threading and context
    • Semantic similarity search
    • Memory compression and summarization
    • Context window management

    üîå Backend Flexibility¶

    Start with SQLite for development, scale to PostgreSQL for production, add Redis for caching, and integrate vector databases for RAG‚Äîall without changing your application code.

    üì¶ Batteries Included¶

    • Caching: Multi-level caching (in-memory + Redis)
    • Backup: Export to JSON, CSV, Excel
    • Analytics: Conversation patterns, usage metrics
    • Migrations: Automated schema management with Alembic

    üöÄ Production Ready¶

    • Comprehensive test coverage (>90%)
    • Connection pooling and retry logic
    • Environment-based configuration
    • Fallback chain support
